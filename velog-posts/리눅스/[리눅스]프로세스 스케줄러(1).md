
- 프로세스 스케줄러(이하 스케줄러)학습에 들어가기 앞서 알아야하는 기본 개념

**경과 시간**: 프로세스 시작부터 종료할 때까지 경과한 시간
**사용 시간**: 프로세스가 실제로 논리 CPU를 사용한 시간

### 경과 시간과 사용 시간을 시각적으로 확인하는 실습 1.
```bash
#!/bin/bash

MULTICPU=0
PROGNAME=$0
SCRIPT_DIR=$(cd $(dirname $0) && pwd)

usage() {
    exec >&2
    echo "사용법: $PROGNAME [-m] <프로세스 개수>
    일정 시간 동작하는 부하 처리 프로세스를 <프로세스 개수>로 지정한 만큼 동작시켜서 모두 끝날 때까지 기다립니다.
    각 프로세스 실행에 걸린 시간을 출력합니다.
    기본값은 모든 프로세스가 1개의 논리 CPU에서 동작합니다.

옵션 설명:
    -m: 각 프로세스를 여러 CPU에서 동작시킵니다."
    exit 1
}

while getopts "m" OPT ; do
    case $OPT in
        m)
            MULTICPU=1
            ;;
        \?)
            usage
            ;;
    esac
done

shift $((OPTIND - 1))

if [ $# -lt 1 ] ; then
    usage
fi

CONCURRENCY=$1

if [ $MULTICPU -eq 0 ] ; then
    # 부하 처리를 CPU0에서만 실행시킴
    taskset -p -c 0 $$ >/dev/null
fi

for ((i=0;i<CONCURRENCY;i++)) do
    time "${SCRIPT_DIR}/load.py" &
done

for ((i=0;i<CONCURRENCY;i++)) do
    wait
done

```

**사용법**:  ./multiload.sh [프로세스 개수]
- 일정 시간 동작하는 부하 처리 프로세스를 프로세스 개수로 지정한 만큼 동작시켜서 모두 끝날 때까지 기다립니다.
- 각 프로세스 실행에 걸린 시간을 출력합니다.
- 기본값은 모든 프로세스가 1개의 논리 CPU 에서 동작합니다.
```bash
sok@controller:~/examples/03$ ./multiload.sh 1

real	0m1.137s # 경과 시간
user	0m1.128s # 사용 시간
sys		0m0.008s # 유휴 시간

sok@controller:~/examples/03$ ./multiload.sh 2

real	0m2.260s
user	0m1.125s
sys		0m0.004s

real	0m2.334s
user	0m1.202s
sys		0m0.000s

sok@controller:~/examples/03$ ./multiload.sh 3

real	0m3.406s
user	0m1.136s
sys	0m0.000s

real	0m3.421s
user	0m1.135s
sys	0m0.004s

real	0m3.507s
user	0m1.225s
sys	0m0.000s

```

실습 결과: 
동시 실행을 2배, 3배로 바꾸더라도 개별 프로세스의 사용 시간은 거의 변하지 않지만, 전체 실행 시간은 2배, 3배 가까이 늘어났습니다. 논리 CPU는 한 번에 프로세스 하나만 처리할 수 있으므로, 스케줄러가 각 프로세스에 순서대로 CPU 자원을 할당하기 때문에 전체 실행 시간은 프로세스 개수에 비례합니다.

### 경과 시간과 사용 시간을 시각적으로 확인하는 실습 2.

논리 CPU 여러 개를 사용하는 경우
multiload.sh 프로그램에 -m 옵션을 지정해서 실행하면 스케줄러가 부하 처리를 모든 논리CPU에 균등하게 배분.

```bash
sok@controller:~/examples/03$ ./multiload.sh -m 1

real	0m1.287s
user	0m1.279s
sys		0m0.008s
sok@controller:~/examples/03$ ./multiload.sh -m 2

real	0m1.119s
user	0m1.112s
sys		0m0.004s

real	0m1.189s
user	0m1.184s
sys		0m0.004s
sok@controller:~/examples/03$ ./multiload.sh -m 3

real	0m1.570s
user	0m1.082s
sys		0m0.008s

real	0m1.737s
user	0m1.118s
sys		0m0.000s

real	0m1.838s
user	0m1.360s
sys		0m0.000s

```
실습 결과: 모든 프로세스에서 real과 user+sys 값이 거의 같습니다. 즉, 프로세스마다 각각의 논리 CPU 자원을 독점했다는 뜻입니다.


* 직감적으로는 언제나 real >= user + sys 일 것이라 느껴지지만 꼭 그렇지만은 않다. 
(각각 시간 측정 방법이 다르고, 
측정 정밀도가 높지 않고, 
자식 프로세스를 생성하여 각각 다른 논리 CPU에서 동작한 경우 특히 그러함.)
